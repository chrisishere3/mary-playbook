## What it is

Conversations with API integration partners — companies whose users we can serve through their existing software (DoorLoop, Rentec Direct, Elise AI, etc.).

Different motion entirely from PM-facing outbound. PM-facing sells one PM at a time. Partner deals touch many PMs or renters at once through a single signed agreement. Bigger leverage per signed partner, slower sales cycle, different buyer (partnerships / BD / product, not ops).

## How it actually works

The proven path is **trigger-based**, not outbound-driven:

1. **Trigger.** Conference floor demo, warm intro from an existing partner, or inbound interest.
2. **Ben takes the intro call.** Establishes commercial alignment at the executive level. Frames it as ARPU expansion for the partner.
3. **Darius takes over.** As soon as Ben finds commercial fit with the partner's exec, Darius (product/engineering) runs the technical discovery and integration conversation. He talks to the partner's product and engineering counterparts. This handoff has been working well.
4. **Integration tier choice.** Link-out → co-branded → embedded iframe (4 API calls). Most partners start at link-out and expand once revenue shows up.
5. **Agreement + rev share doc.**
6. **Engineering.** 3 days via CSV. 4 API calls via iframe. Live in weeks, not months.

## What's been tried, what hasn't

**Worked:**

- Warm intros from existing partners
- Conference floor demos that progress to formal Ben + Darius calls

**Didn't work:**

- A cold-outbound sprint to partners. Tested lightly, no real traction. Not a proven motion for this segment.

## Proof we lean on

- **TenantCloud added $1M+ in annual revenue.** 3-day implementation via CSV. Mark DeHaan quote on file.
- **TurboTenant** ran on the simplest link-out integration for a year — we paid them more than the partner they were promoting.

## Who's in the bullseye

- **Mid-market PMS without a native utility product:** DoorLoop, Rentec Direct, Innago, Hemlane, Avail, Stessa, RentRedi
- **AI leasing tools:** Elise AI, Funnel, Zuma, BetterBot, Quext
- **Tenant screening:** RentSpree, Snappt, Findigs
- **Rental marketplaces:** Zumper, Rent., Redfin Rentals

## Who's blocked

- **RealPage / Updater-stack platforms:** Yardi, AppFolio Enterprise, Entrata, ResMan, MRI, AMSI, Knock
- **Updater itself, Moved, Porch** — direct competitors

## Where there's more room

- **Inbound from this segment.** Conversations come in unprompted more than they're chased. A lightweight "partner with us" page or form could let inbound qualify itself.
- **A second case study at TenantCloud's scale.** TenantCloud's $1M+ ARR is doing a lot of heavy lifting. A second one lifts close rates across the bullseye list.
- **Deck for screen-share on Ben + Darius calls.** Today the pitch is verbal. A short deck with the TenantCloud case study and the integration-tier diagram would help.
- **TurboTenant on the record.** They've validated the model but haven't given us a public quote yet.
- **A repeatable cold outbound motion** has not been figured out for this segment. If you want one, it's a real zero-to-one project. The current proven path doesn't require it.
